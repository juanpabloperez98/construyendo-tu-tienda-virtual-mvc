1.2 ¿Qué es el patrón MVC?

El patrón de diseño Modelo-Vista-Controlador (MVC) es una arquitectura ampliamente utilizada en el desarrollo de software, especialmente en aplicaciones web y móviles. Su principal objetivo es dividir el código de la aplicación en tres componentes interconectados pero con responsabilidades claramente definidas: el Modelo, la Vista y el Controlador. Esta separación facilita la modularidad, mejora el desarrollo colaborativo y hace que el código sea más fácil de mantener y escalar.

El Modelo maneja la lógica de negocios y los datos, interactuando con bases de datos o fuentes externas para consultar, modificar o eliminar información. En una aplicación de comercio electrónico, podría incluir clases como Producto, Usuario o Pedido.
La Vista es la interfaz de usuario, encargada de mostrar los datos y capturar interacciones. Se compone de HTML, CSS y scripts que presentan la información de manera visual y amigable.
El Controlador actúa como intermediario entre la Vista y el Modelo. Procesa las acciones del usuario, interactúa con el modelo para obtener o modificar datos y actualiza la vista con la información

Modelo (Model)

Definición: Representa la lógica de negocio y los datos de la aplicación. Gestiona y almacena información, y contiene las reglas, lógica y funciones específicas que procesan los datos.

Responsabilidades: Interacción con bases de datos (CRUD: crear, leer, actualizar, eliminar), procesamiento, validación de datos, y ejecución de reglas y operaciones de negocio.

Vista (View)

Definición: Es la representación visual de la información, es decir, todo lo que el usuario experimenta visualmente e interactúa con la interfaz de la aplicación.

Responsabilidades: Mostrar datos al usuario, recibir la interacción del usuario (por ejemplo, clicks, entradas de teclado, etc), y presentar la información de una manera comprensible y atractiva.

Controlador (Controller)

Definición: Actúa como intermediario entre el Modelo y la Vista. Responde a las interacciones del usuario y realiza llamadas al Modelo y la Vista según sea necesario.

Responsabilidades: Gestionar las entradas del usuario, solicitar al Modelo que realice ciertas operaciones o cambios, y actualizar la Vista en consecuencia.

1.3 Ejemplos de MVC en acción

El patrón Modelo-Vista-Controlador (MVC) es una poderosa estructura utilizada en el desarrollo de software que, cuando se implementa correctamente, proporciona claridad, separación de preocupaciones y facilita la mantenibilidad. Pero, ¿cómo se ve esto en la práctica real? Para comprender mejor cómo interactúan estos tres componentes y cómo se combinan para crear aplicaciones robustas y coherentes, vamos a explorar algunos ejemplos concretos de cómo se aplicaría el MVC en escenarios del mundo real.

A continuación, puedes continuar con los ejemplos detallados, como el de la Red Social que se va a mencionar o cualquier otro ejemplo que desees incluir para ilustrar el funcionamiento del patrón MVC.

Aplicación de tareas pendientes (To-Do List)

Modelo: Almacena la lista de tareas, cada tarea con su estado (pendiente, completada).

Vista: Muestra la lista de tareas al usuario, permitiéndole marcar tareas como completadas o añadir nuevas tareas.

Controlador: Cuando un usuario añade una tarea, el Controlador la procesa, informa al Modelo para que la almacene y actualiza la Vista para mostrar la nueva tarea.

Red Social

Imaginemos una plataforma similar a Facebook o Twitter donde los usuarios pueden publicar actualizaciones de estado, seguir a otros usuarios, y recibir noticias o actualizaciones de aquellos a quienes siguen.

Modelo: Almacena información de cada usuario, como nombre y correo electrónico, junto con sus publicaciones y actualizaciones, incluyendo fechas y detalles. Además, registra las relaciones de seguimiento entre ellos.

Vista: Exhibe detalles del perfil, las últimas publicaciones, seguidores y seguidos; muestra las actualizaciones recientes de conexiones y facilita el registro e ingreso a la plataforma.

Controlador: Cuando un usuario publica una actualización, el Controlador procesa la solicitud, informa al Modelo para almacenarla y luego actualiza la Vista para reflejar la nueva entrada tanto en el perfil como en el feed de noticias.

¿Qué es un modelo en SQLAlchemy ORM?

Un modelo en SQLAlchemy ORM es una representación en código de una tabla en una base de datos. Cada modelo se define como una clase en Python, donde los atributos de la clase corresponden a las columnas de la tabla. Esto permite interactuar con la base de datos usando objetos y métodos en lugar de escribir consultas SQL manualmente.

Definición concreta y sencilla:

Un modelo en SQLAlchemy ORM es una clase en Python que define la estructura de una tabla en una base de datos. Cada instancia de esta clase representa una fila de la tabla, y sus atributos representan las columnas.

1.5 Flujo de Datos en MVC

El patrón Modelo-Vista-Controlador (MVC) es reconocido por la clara delimitación de responsabilidades y el flujo estructurado de datos entre sus tres componentes principales. A continuación, desglosaremos este flujo para entender cómo la información y las responsabilidades se mueven entre el Modelo, la Vista y el Controlador.

Solicitud del Usuario: Todo comienza cuando el usuario realiza una acción o solicitud en la interfaz de la aplicación (Vista). Esto podría ser algo como hacer clic en un botón, llenar un formulario o navegar a otra página.

Intervención del Controlador: La Vista comunica esta acción al Controlador correspondiente. El Controlador interpreta esta acción y decide qué hacer con ella. Si se requieren datos o algún tipo de procesamiento, el Controlador solicitará al Modelo la información o acción necesaria.

Procesamiento del Modelo: El Modelo, que encapsula la lógica de negocio y la gestión de datos, procesa la solicitud del Controlador. Esto puede involucrar consultar una base de datos, realizar cálculos, o cualquier otro tipo de procesamiento necesario. Una vez que ha finalizado, devuelve cualquier dato o resultado relevante al Controlador.

Actualización de la Vista: El Controlador, tras recibir la respuesta del Modelo, procede a actualizar la Vista con la información adecuada. Como por ejemplo mostrar nuevos datos, presentar un mensaje al usuario, o incluso redirigir a otra parte de la aplicación.

Respuesta al Usuario: Finalmente, la Vista se actualiza de acuerdo a las instrucciones del Controlador, presentando al usuario el resultado de su solicitud inicial o mostrando la información actualizada.

El flujo de MVC, en esencia, es un ciclo constante de acciones del usuario, procesamiento de datos y actualizaciones de la interfaz. La belleza de este patrón radica en la separación de responsabilidades, permitiendo que cada componente se enfoque en su tarea específica y se pueda modificar de forma independiente sin afectar a los demás.
